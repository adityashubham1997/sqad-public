🏛️ Architecture Layers

How the framework components stack together

CLI Layer — bin/sqad-public.js

Entry point. Parses commands, dispatches to lib modules. npx sqad-public init

Detection Layer — lib/detect/

stack.js + cloud.js + tracker.js — auto-detect everything about the workspace

Generation Layer — lib/generate/

config.js + context.js + ide-skills.js — generate config, context files, and IDE-specific skills

Transform Layer — lib/transform/

8 IDE transformers: Claude, Windsurf, Cursor, Codex, Kiro, Gemini, AntiGravity, KG tools

Agent Layer — sqad-method/agents/

16 agents with unique personalities, review lenses, capabilities. Extends _base-agent.

Skill Layer — sqad-method/skills/

27 phase-gated skills orchestrating multi-agent workflows (dev-task, review, QA, AI audit, etc.)

Fragment Layer — sqad-method/fragments/

35+ modular knowledge units: stack (20), cloud (5), core (7), tracker (3), rubric (18+)

Knowledge Graph — lib/kg/

build.js + summary.js — extract and organize project context into queryable graphs
